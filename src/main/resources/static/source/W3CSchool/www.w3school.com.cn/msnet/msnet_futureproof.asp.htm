下面的段落描述了 W3School 对未来的因特网分布式应用程序的展望。

应用程序应当支持未来的变化

应用程序应当能意识到未来的变化，并良好地适应服务数量以及交换格式方面的变化。应用程序和数据交换格式必须支持扩展和已删除元素及版本控件。我们的建议是最好使用带有 XML Schema 的 XML。

应用程序应当可伸缩并可重新部署

应用程序应当有能力弹性地支持每日少许直到千百万次的请求。服务器服务应当有能力分布到多个服务器，或者在不中断应用程序的情况下从一个服务器转移至另外一个。我们的建议是使服务器服务独立于物理服务器，并使用标准的因特网统一资源定位器 (URL) 或者目录服务来定位服务。

应用程序必须支持其他应用程序

应用程序应当有能力与其它应用程序协同工作。我们的建议是最好使用相同的规则来创建所有的应用程序，并使用标准的因特网链接来连接这些应用程序。

应用程序必须支持未来的因特网标准

应用程序应当做好准备来支持新的因特网标准。
