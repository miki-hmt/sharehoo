deferred.isResolved()1.8-

概述    deferred.isResolved()

返回值：Deferred Object
确定延迟对象是否已得到解决。

描述： 确定一个Deferred（延迟）对象是否已被解决。

  • V : 1.5deferred.isResolved()

    • 这个方法不接受任何参数

从jQuery1.7开始，这个 API已经过时; 请使用deferred.state() 代替。

如果Deferred（延迟）对象是在被解决的状态则返回true，这意味着要么deferred.resolve()或者deferred.resolveWith()已经在对象上被调用过，并且 failCallbacks 已经被调用过（或者正处于被调用的阶段）。

请注意，Deferred（延迟）对象可以有三种状态：挂起（pending），解决（resolved），或拒绝（rejected）;使用deferred.isRejected()来判断延迟对象是否在拒绝状态。这些方法主要用于调试，例如，在准备拒绝（reject）一个延迟对象前，判断它是否已经处于 resolved 状态。
